# Plan — coffee-bet

`.claude/plan/` 은 **"지금 뭘 어떤 순서로 부딪힐지"** 를 정리하는 작업 공간이다.
큰 그림 명세는 `../project-spec.md` 와 `../architecture/` 가 가지고 있고, 여기서는 "다음 한 발"을 본다.

## 문서 구성

| 파일 | 역할 |
|---|---|
| `todo.md` | 마일스톤별 체크리스트. 진행 중인 단계는 세분화, 나머지는 큰 덩어리. |
| `requirements-m1.md` | 지금 단계 (M1) 한정 수용 기준(Definition of Done) + 입출력 명세. |

## 흐름

1. **`requirements-m1.md` 가 먼저** — "M1 끝났다고 부를 수 있는 기준" 을 박아둔다. 이게 흐릿하면 todo 도 흐릿해진다.
2. **`todo.md` 는 매일/매 작업 단위 갱신** — 끝낸 항목은 `[x]` 로 체크. 새로 발견한 작업은 추가.
3. M1 끝나면:
   - `requirements-m1.md` 를 `requirements-m1-done.md` 로 옮기거나 archive
   - `requirements-m2.md` 새로 작성
   - todo 의 M2 섹션을 세분화

## 학습 모드 규칙 (자기 지침)

- 코드는 직접 작성. AI 한테는 *왜?*, *트레이드오프*, *디버깅 가설* 만 묻는다.
- 막히면 todo 항목 옆에 "❓ 질문할 것" 으로 메모해두면, 나중에 한꺼번에 AI 에 던지기 좋음.
- 한 마일스톤 안에서는 가능한 **돌아가는 상태** 를 유지한다 (`go run` 하면 적어도 컴파일은 된다).
